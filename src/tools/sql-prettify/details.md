# SQL Prettify & Formatter: Writing Better Queries

Structured Query Language (SQL) is the backbone of data management, but as queries grow in complexity, they often become a tangled mess of keywords, table names, and join conditions. A single "wall of text" query is difficult to read, impossible to debug, and a nightmare for peer reviews.

The **Armytool SQL Prettify** tool is designed to bring order to your data logic.

## 1. What the Tool Does
The SQL Prettify tool is an intelligent formatter that understands SQL syntax across multiple dialects (MySQL, PostgreSQL, SQL Server, etc.). It transforms cramped queries into structured, readable code by:
- **Keyword Alignment:** Standardizing the placement of `SELECT`, `FROM`, `WHERE`, `GROUP BY`, and other major clauses.
- **Indentation:** Nesting subqueries and `JOIN` conditions for visual clarity.
- **Casing:** Automatically converting keywords to uppercase for better contrast against identifiers.
- **Comment Preservation:** Keeping your inline documentation intact while cleaning up the surrounding code.

## 2. Why Professionals Use It
Readable SQL isn't just about aesthetics; it's a technical requirement for high-performing teams:
- **Easier Debugging:** When each clause is on a new line, finding a missing comma or a logic error in a `WHERE` condition takes seconds instead of minutes.
- **Better Peer Reviews:** Clean code is easier for your teammates to understand and approve.
- **Code Consistency:** Ensures that every query in your project's codebase follows the same stylistic standards.
- **Documentation:** Prettified SQL is much more professional when included in technical specs, Jira tickets, or blog posts.

## 3. Step-by-Step Instructions

1. **Input:** Paste your raw, unformatted SQL query into the text area.

## 4. Examples

### Messy Input
`select u.id,u.name,o.total from users u join orders o on u.id=o.user_id where o.status='shipped' order by o.total desc`

### Prettified Output
```sql
SELECT
  u.id,
  u.name,
  o.total
FROM
  users u
  JOIN orders o ON u.id = o.user_id
WHERE
  o.status = 'shipped'
ORDER BY
  o.total DESC
```

## 5. FAQs

**Q: Does this tool support my specific database?**
A: Yes, our formatter supports standard SQL as well as dialects like MySQL, MariaDB, PostgreSQL, SQL Server, and Oracle.

**Q: Can it fix my SQL errors?**
A: No. This is a **formatter**, not a linter or compiler. Your SQL must be syntactically valid (even if messy) for the tool to format it correctly.

**Q: Is my query data secure?**
A: 100%. Like all Armytool utilities, the SQL Prettify tool runs **entirely in your browser**. Your table names and data never leave your computer.

## 6. Common Mistakes
- **Formatting Invalid SQL:** If the output looks strange, double-check that you didn't miss a closing parenthesis in a subquery.
- **Over-Minifying:** While minified SQL saves a few bytes in a network request, it should never be used in your source code or documentation.
- **Ignoring Casing:** Mixed-case keywords make SQL much harder to scan. Always use the "Uppercase Keywords" feature.
